google搜索技巧

发表于 2016-03-28 | 分类于 发现共享 |

在大学折腾因特网两年，发现Google真的是个好东西，然而现在在国内无法访问，真是可惜。但是大家还是要学会科学上网，因为这将给你的上火带来巨大的便利。以下是我在使用Google过程所总结的一些技巧，或许会对大家对互联网的探索增加了方便。

基本搜索: +, -, OR

  • GOOGLE无需用明文的“+”来表示逻辑“与”操作，只要空格就可以了。
    示例：搜索所有包含关键词“易筋经”和“吸星**”的中文网页
  • GOOGLE用减号“-”表示逻辑“非”操作。
    示例：搜索所有包含“易筋经”而不含“吸星++”的中文网页
GOOGLE用大写的“OR”表示逻辑“或”操作。但是，关键字为中文的或查询似乎还有BUG，无法得到正确的查询结果。

示例：搜索包含布兰妮“Britney”或者披头士“Beatles”、或者两者均有的中文网页。

辅助搜索：通配符、大小写、句子、忽略字符以及强制搜索

  • “ ”以整个短语作为搜索关键词，而不是拆开成每个词
    示例：搜索包含“long, long ago”字串的页面。
    搜索：“”long, long ago””

高级搜索：site，link，inurl，allinurl，intitle，allintitle

  • “site”表示搜索结果局限于某个具体网站或者网站频道，如“sina.com.cn”、“edu.sina.com.cn”，或者是某个域名，如“com.cn”、“com”等等。如果是要排除某网站或者域名范围内的页面，只需用“-网站/域名”。
  • 示例：搜索中文教育科研网站（edu.cn）上所有包含“金庸”的页面。
    搜索：“金庸 site:edu.cn”
  • 注意：site后的冒号为英文字符，而且，冒号后不能有空格，否则，“site:”将被作为一个搜索的关键字。此外，网站域名不能有“http”以及“www”前缀，也不能有任何“/”的目录后缀；网站频道则只局限于“频道名.域名”方式，而不能是“域名/频道名”方式。诸如“金庸 site:edu.sina.com.cn/1/”的语法是错误的。

inurl语法返回的网页链接中包含第一个关键字，后面的关键字则出现在链接中或者网页文档中。有很多网站把某一类具有相同属性的资源名称显示在目录名称或者网页名称中，比如“MP3”、“GALLARY”等，于是，就可以用INURL语法找到这些相关资源链接，然后，用第二个关键词确定是否有某项具体资料。INURL语法和基本搜索语法的最大区别在于，前者通常能提供非常精确的专题资料。

  • 示例：查找微软网站上关于windows2000的安全课题资料。
    搜索：“inurl:security windows2000 site:microsoft.com”

注意：“inurl:”后面不能有空格，GOOGLE也不对URL符号如“/”进行搜索。GOOGLE对“cgi-bin/phf”中的“/”当成空格处理。

PDF文档搜索

我对GOOGLE尤其欣赏的一点就是它提供对PDF文档内文的检索。目前GOOGLE检索的PDF文档大约有2500万左右。这真是太美妙了。PDF是ADOBE公司开发的电子文档格式，现在已经成为互联网的电子化出版标准。PDF文档通常是一些图文并茂的综合性文档，提供的资讯一般比较集中全面。

  • 示例：搜索关于电子商务（ECOMMERCE）的PDF文档。
    搜索：“inurl:pdf ecommerce”
  • Filetype: 只搜索指定类型的文档，可以用来搜索pdf,doc,jpg等类型的文档

招投标工程总结

发表于 2016-03-24 | 分类于 技术总结 |

工程实践2感悟

  • 写代码的过程中要实时写注释，不要拖到工程完成时才来写注释，这时可能前面写的代码当时的思路已经忘记了，注释的目的为了以后自己重构时方便理解，同时也给其他开发者能轻松读懂你的代码
  • 在写代码开始前，对整个工程的设计很重要，这关系到后面写代码的困难与否。你设计时想的越透彻后面写代码时就越轻松。
  • 注重考虑用户的感受，我觉得软件是做给用户用的，就必须让用户用起来很方便，没有什么学习软件的成本。如果是大型软件，就应该给出用户手册，尽量减少用户学习软件的时间。细节方面我觉得最为重要，往往细节决定成败，这在软件上表现的更明显。细节对用户的心里感受最明显。体现的方面有软件的美工，布局，逻辑。或许你觉得我实现了所有功能，可是你没有实现用户想要的那个功能就是耍流氓。

  • 对于异常的处理，在认为有可能出现的异常的地方都要加上异常的处理，这体现代码的安全性，其实也是对用户体验的考虑，c#如果你未进行异常处理的时候出现了异常，这是.net会把错误抛给操作系统，操作系统一般会终结这个程序，这大大降低了用户的体验。

  • 实时思考，想到了就去实现，不然会忘的。

杂感

发表于 2016-02-27 | 分类于 日记 |

回望过去

　　听到一段旋律，莫名悲伤起来，会勾起过去的记忆。时光荏苒，转眼也快毕业了，能学习的时间也就半个学期了。还记得当初自己一个人背着行囊来到这个陌生城市求学的样子，那是我第一次出远门。时光易逝，光阴不减。现在我的生活忙碌而充实，为了自己不曾企及的梦想。考研，在我大一大二看来真的是遥不可及，那时候我正忙于参加各种活动，被大学丰富的生活遮蔽了双眼。也许是以前的生活太压抑了吧，大学里大家都放松了，然而真的我们就在最需要奋斗的年纪选择了安逸，这也不能怪我们，因为以前真的是太累了。遥想高中，似乎真的槽糕的不行，初中以很好的成绩考入我们县城的高中，然而随后却没能取得我想要的成功，获得的却是失败。当你一开始就没能跟上节奏，后来就再难以追赶，因为你最初的方法都是错的，怎么可能追的上！知道这些时，我的高中生涯已经结束了。
　　近期考研都在有条不紊的进行着，高数复习到难的那一部分了，现在的计划是能在五一前把数学过一遍，恋恋有词和外刊精读看完。快十二点，马上滚去睡觉了。突发奇想写了这篇文章，希望以后能每天周写一篇总结下生活和学到的技术。

故乡

发表于 2016-02-27 | 分类于 生活随笔 |

叶落归根

我是向着远方独行的浪子，年少时想仗剑走天涯，看一看世界的繁华。当你离开故乡，走入这花花世界以后，仍然觉得故乡始终是剪不断，理还乱。或许每个人背井离乡的原因有很多种，但是回到故乡只有一种，因为那是你的根，你是喝着那里的水长大的，可以说故乡赋予了你这般体魄同时你的魂也牵挂着故乡。
家乡的天气真好

以前没有离开的时候，觉得家里很不好，但是离开后才发现故乡才是最可爱的。现在我仍然记得家里的四季的样子，生机勃勃的春天，湿热好玩的夏天，美丽迷人的秋天，万物寂静的冬天。四季分明，每一季都有好玩的好吃的。且不说春天里满山的杜鹃，还有那充满野味的蒿子粑。故乡在小时候给我无论视觉、味觉的冲击太大了，似乎每天都是新的一天。那时候才是最快乐的时光，真的叫无忧无虑。我和小伙伴们一起上山采花、采蘑菇，去池塘里抓蝌蚪、抓鱼。放学了就一起玩各种游戏，都是自己创造的游戏，玩法很独特，真的比现在的孩子抱着Ipad玩游戏有趣多了。
后来随着进入初中、高中，回家的次数越来越少，但好歹还是偶尔会回去，故乡的四季一直没缺席过，虽然那时已无心欣赏这美色(面临升学的压力)。到现在上了大学，只有寒暑假回去一次，故乡再无春秋，只有冬夏。每每从家里去学校，都会带上故乡的味道，希望她能留的更久一点。或许以后回去的时间越来越少了，可是记忆中的故乡还是那样，不会有任何改变。

过年回去了一趟，样子还是那个样子，但是觉得故乡没那个精神气了，仿佛像暮年的老人，印象中的故乡明明是个小伙儿。城镇化的脚步叶走进了我们那个偏远的村庄，乡镇盖起了大楼，路也好了不少，但是村的人却越来越少，大家也没以前那么亲热了，过年相互串门的都不多。要记得我小时候过年真的特别热闹，家家都会走动，而且也没有院墙。东家的炸圆子送给了西家，隔壁家的王大爷叫我们去尝尝他的手艺，现在这些都没有了。这让我有些失落，可我也明白，这是中国城镇化的一个阵痛，想必西方应该也有吧。
时光时光慢些吧，不要让她变老。
